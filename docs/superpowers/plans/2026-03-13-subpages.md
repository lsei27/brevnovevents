# Subpages Implementation Plan: /firemni-eventy & /svatba-v-klastere

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build two conversion-focused subpages for Břevnovský klášter event venue — a corporate events detail page and a wedding page with alternate color scheme.

**Architecture:** Both pages follow Next.js App Router conventions. Each page gets its own route directory, page.tsx with metadata, and composes reusable + page-specific section components. The wedding page uses pink/gold accent colors instead of red. Both pages reuse existing shared components (Button, ImageCarousel, Header/Footer, Location, Contact) and share the existing form infrastructure with a new Wedding Form B variant.

**Tech Stack:** Next.js 15 (App Router), Tailwind CSS v4, TypeScript strict, Zod validation, Server Actions

import { z } from "zod";

export const weddingSchema = z.object({
  name: z.string().min(1, "Toto pole je povinné."),
  email: z.string().email("Zadejte platnou e-mailovou adresu."),
  phone: z.string().min(1, "Toto pole je povinné."),
  preferredDate: z.string().min(1, "Toto pole je povinné."),
  guestCount: z.string().min(1, "Toto pole je povinné."),
  celebrationType: z.string().min(1, "Vyberte alespoň jednu možnost."),
  catering: z.string().min(1, "Toto pole je povinné."),
  vision: z.string().optional(),
  source: z.string().optional(),
  website: z.string().optional(),
});

export type WeddingFormData = z.infer<typeof weddingSchema>;

export const weddingGuestCountOptions = [
  "Do 30", "30–60", "60–100", "100–150", "150–200", "Nad 200",
];

export const celebrationTypeOptions = [
  "Venkovní obřad", "Slavnostní hostina", "Cocktail reception", "Afterparty", "Celý svatební den",
